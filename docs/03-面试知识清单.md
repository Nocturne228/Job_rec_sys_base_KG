# 推荐系统面试知识清单

> 从 `interview/` 目录中 ByteDance 专项面试准备和旧版简历差异应对中提炼的核心知识点。
> 用于面试前快速自检知识覆盖度。

---

## 一、推荐系统基础架构

### 工业推荐系统标准范式

- [ ] 召回→粗排→精排→重排 四级架构及各层职责
- [ ] 召回求快（百万→千）、排序求准（千→十）、重排求生态
- [ ] 北极星指标：推荐投递 CVR（非 CTR）
- [ ] 离线指标：Recall@K, NDCG@K, HitRate@K, MRR, GAUC
- [ ] 离线到线上映射：CTR~Recall×倾向因子, CVR~CTR×转化率

### 特征体系

- [ ] 用户画像特征（年龄/性别/地域/设备/长期兴趣）
- [ ] 用户行为序列（最近 N 个交互 ID 序列 → DIN/DIEN 建模）
- [ ] 物品内容特征（标题/标签/多模态 Embedding）
- [ ] 统计特征（历史 CTR/完播率/点赞率）
- [ ] 交叉特征（用户兴趣 × 物品标签匹配度）
- [ ] 上下文特征（时间/网络/位置）
- [ ] 实时特征管线：Kafka→Flink 滑动窗口→Redis→模型读取
- [ ] training-serving skew 的原因和避免方法

## 二、召回层

### 多路召回体系

- [ ] i2i 协同过滤（ItemCF: 共现矩阵）
- [ ] u2u 协同过滤（UserCF: 相似用户→推荐）
- [ ] Embedding 召回（双塔 DSSM + ANN 检索）
- [ ] 图召回（LightGCN / GraphSAGE / PinSAGE）
- [ ] 多兴趣召回（MIND: Capsule Network 拆解 K 个兴趣向量）
- [ ] 热门/趋势召回、地理召回、社交召回
- [ ] 探索召回（ε-greedy / Thompson Sampling）

### 双塔 DSSM

- [ ] 用户 Tower + 物品 Tower 独立编码 → 内积打分
- [ ] 推理可分离性：物品侧离线预计算，用户侧实时编码
- [ ] 负采样策略：in-batch negative / 全局 negative (MoCo Queue) / hard negative
- [ ] InfoNCE Loss / 对比学习

### 图召回

- [ ] LightGCN：去 W 去 σ，纯邻居聚合，BPR Loss
- [ ] PinSAGE：GraphSAGE + 注意力聚合 + 邻居采样
- [ ] 大规模图：图分区 + 邻居采样(k-hop 邻居爆炸) + 增量更新

## 三、排序层

### CTR 预估模型演进

- [ ] LR (手工特征交叉) → Wide&Deep (记忆+泛化)
- [ ] DeepFM (FM 自动二阶交叉 + DNN 隐式高阶)
- [ ] xDeepFM (CIN 显式高阶交叉)
- [ ] AutoInt (自注意力特征交叉)

### 行为序列建模

- [ ] DIN: Local Activation Unit — 候选相关注意力提取历史兴趣
- [ ] DIEN: AUGRU 建模兴趣时序演化
- [ ] BST: Transformer 行为序列
- [ ] DIN vs Transformer Attention 的区别 (单向 vs 双向, O(seq×d) vs O(seq²×d))

### 多目标优化

- [ ] MMOE: Multi-gate Mixture of Experts — 多任务共享+门控
- [ ] PLE: 渐进式分层 — 任务专属 Expert + 共享 Expert
- [ ] ESMM: 解决 CVR 样本选择偏差(SSB) — 全空间 pCTCVR=pCTR×pCVR
- [ ] 任务冲突：CTR 要强刺激 vs 完播率要叙事连贯

### 其他排序要点

- [ ] 线性融合 vs DeepFM 的权衡（数据量/可解释性/延迟）
- [ ] 特征交叉的重要性：单一特征信息有限，交叉信号是核心
- [ ] GAUC vs AUC：按用户分组加权，更公平评估个性化排序

## 四、重排层

- [ ] 多样性打散（MMR: α·relevance + (1-α)·diversity）
- [ ] 新鲜度控制（新内容加权 + 时间衰减）
- [ ] 探索与利用（ε-greedy / Thompson Sampling / UCB）
- [ ] 业务规则（同类去重/创作者扶持/广告混排/审核安全）
- [ ] Filter bubble / information cocoon 的诊断与修复

## 五、在线学习

- [ ] FTRL: 在线增量学习 + L1 稀疏 (百万特征大部分权重为 0)
- [ ] 离线日更深度模型 + 在线 FTRL 分钟级更新的混合模式
- [ ] 样本回流闭环：在线推荐→用户交互→Kafka→Flink→样本库→离线训练
- [ ] 模型热切换：在线 FTRL 权重定期 Dump→离线模型初始化

## 六、A/B Testing

- [ ] 实验设计：控制组 vs 实验组，50%/50%，≥14 天
- [ ] 样本量计算：`statsmodels.stats.power.NormalIndPower`
- [ ] CVR 检验：z-test for proportions
- [ ] 均值检验：Welch's t-test
- [ ] 非正态检验：Mann-Whitney U
- [ ] 置信区间：Bootstrap CI
- [ ] 陷阱：新奇效应(Novelty Effect)、学习效应、辛普森悖论、外部因素

## 十二、系统设计速查

**"如何设计抖音推荐系统" 回答框架**:
1. 北极星指标：人均使用时长 + 留存率
2. 召回层：10+ 路召回（标签/CF/Embedding/MIND/社交/热门/探索）
3. 粗排层：双塔 DSSM 2000→200
4. 精排层：MMOE/PLE 多目标(CTR/完播/点赞/评论/转发/关注) + DIN/DIEN 行为序列
5. 重排层：MMR 多样性 + 新鲜度 + E&E + 广告混排
6. 在线学习：FTRL 分钟级 + 离线日更深度模型
7. A/B 实验：小流量验证→逐步放量 5%→10%→50%→100%
