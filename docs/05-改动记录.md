# 赛题合规改动记录

> 基于服创大赛 A15 赛题需求评估，对 JobRec_KG 项目进行的全量改造。
> 本次改动使项目从"面试作品集 PoC"升级为"赛题可交付系统"。

## 改动动机

在对 `ref/01-参考文献总览.md`、`ref/03-文献综合分析.md` 中的研究计划与赛题需求进行交叉评估后，发现以下关键缺口：

1. **FR-7（招聘推荐）**：赛题明确要求"企业招聘人员输入职位要求→匹配求职简历"，项目仅有单向推荐
2. **QR-3（隐私加密）**：赛题要求 ≥4 项个人信息加密，项目完全没有实现
3. **FR-4（推荐展示+反馈）**：赛题要求"推荐结果展示+用户反馈+演示视频"，缺乏 Web 交互层
4. **FR-8（热门趋势统计）**：赛题扩展功能要求热门职位/技能/专业趋势图

## 一、新增模块

### 1.1 隐私加密模块 (`src/utils/crypto.py`)

| 能力 | 实现 | 覆盖赛题需求 |
|------|------|-------------|
| AES-256-CBC 加密 | `encrypt_personal_info()` / `decrypt_personal_info()` | QR-3 ≥4 项加密 |
| PBKDF2 密钥派生 | `_derive_key()` — SHA-256 + 100,000 轮迭代 | 安全存储 |
| 批量字段加密 | `encrypt_user_profile()` — 自动加密 name/phone/email/address | QR-3 4 项字段 |
| 解密授权 | `decrypt_user_profile()` — 仅授权人员可解密 | QR-3 |

### 1.2 数据脱敏模块 (`src/data/anonymizer.py`)

| 能力 | 实现 | 覆盖赛题需求 |
|------|------|-------------|
| 邮箱脱敏 | `anonymize_email()` → `ab***@domain` | QR-3 隐私保护 |
| 手机脱敏 | `anonymize_phone()` → `138****5678` | QR-3 |
| 姓名脱敏 | `anonymize_name()` → `张***` | QR-3 |
| 简历文本脱敏 | `anonymize_resume()` — 正则替换 EMAIL/PHONE/ID_NUMBER | QR-3 |

### 1.3 反向匹配模块 (`src/matching/reverse_matcher.py`)

| 能力 | 实现 | 覆盖赛题需求 |
|------|------|-------------|
| 企业端候选人搜索 | `ReverseMatcher.match_candidates()` | FR-7 招聘推荐 |
| 多因子评分 | `0.6·SkillCoverage + 0.4·SBERT_Similarity` | FR-3 推荐模型 |
| GAT 加权集成 | 复用现有 `SkillCoverageCalculator` + `GATSkillWeighter` | FR-3 |

### 1.4 推荐有效性采集模块 (`src/metrics/effectiveness.py`) ← 第二轮补充

| 能力 | 实现 | 覆盖赛题需求 |
|------|------|-------------|
| 在线反馈收集 | `EffectivenessCollector.record_feedback()` — API 调用触发 | QR-1 |
| 离线模拟测评 | `simulate_effectiveness_from_interactions()` — 从交互数据推断 | QR-1 |
| 统计报告 | `EffectivenessCollector.report()` — 全局 + 分用户有效性 | QR-1 |
| 定义标准化 | 推荐有效性 = N_satisfied / N_total, 目标 ≥ 80% | QR-1 |

### 1.6 趋势分析模块 (`src/analytics/trends.py`)

| 能力 | 实现 | 覆盖赛题需求 |
|------|------|-------------|
| 热门岗位排行 | `hot_jobs()` — 基于交互频率 | FR-8 热门职位趋势 |
| 热门技能统计 | `hot_skills()` + `hot_skills_by_category()` | FR-8 热门技能趋势 |
| 学历分布 | `education_distribution()` | FR-8 热门专业趋势 |
| 经验分布 | `experience_distribution()` — 0-1/1-3/3-5/5+ 年分桶 | FR-8 |
| 综合报告 | `full_report()` — 一键生成所有趋势数据 | FR-8 |

### 1.7 API 交互层 (`src/api/routes.py`)

基于 FastAPI，覆盖赛题全部交互需求：

| 端点 | 方法 | 覆盖赛题需求 |
|------|------|-------------|
| `/api/recommend` | POST | FR-6 岗位推荐（上传简历→推荐职位） |
| `/api/competency` | POST | FR-5 能力评价（简历+期望职位→契合度+差距） |
| `/api/recruit/match` | POST | FR-7 招聘推荐（职位要求→候选人员列表） |
| `/api/feedback` | POST | FR-4 用户反馈（推荐是否有效） |
| `/api/trends/hot-jobs` | GET | FR-8 热门趋势 |

## 五、部署验证

启动 API 服务:
```bash
cd jobrec
uv pip install fastapi uvicorn python-multipart
uvicorn src.api.routes:app --host 0.0.0.0 --port 8000
```

API 文档自动生成: http://localhost:8000/docs

演示视频录制要点（对应 D-4 交付物）:
1. 上传简历 → 调用 `/api/recommend` → 展示推荐列表

3. 输入招聘需求 → 调用 `/api/recruit/match` → 展示候选人排名
4. 调用 `/api/trends/hot-jobs` → 展示热门趋势图表
