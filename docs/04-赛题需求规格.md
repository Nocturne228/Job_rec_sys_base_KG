# 赛题需求规格

> 来源: 中国大学生服务外包创新创业大赛 A15 赛题
> 命题企业: 锐捷网络 | 类别: 智能计算类
> 题目: 基于知识图谱的大学生就业能力评价和职位推荐系统

## 1. 项目背景

2023 届高校毕业生达 1158 万人。当前网络招聘市场职位信息和求职者数量庞大，信息过载导致匹配效率低下。系统需帮助求职者快速找到理想职位、识别技能短板，同时降低企业和求职者的匹配成本。

命题企业（锐捷网络）提供岗位信息样例库（CSV 格式，含专科/本科/研究生）。

## 2. 功能需求

### FR-1 知识图谱构建
构建包含职位、技能要求、行业信息等多领域知识的知识图谱。

### FR-2 用户信息聚合与分析
对用户个人简历、求职意向、工作经验等进行聚合分析，学习用户兴趣和特征，在知识图谱上匹配职位需求。

### FR-3 推荐模型
设计推荐算法和预测模型。可选方向：协同过滤、关键因素模型、深度学习模型等。

### FR-4 推荐结果展示
将推荐结果呈现给用户，支持用户选择和反馈。呈现方式可根据不同用户群体定制。

### FR-5 能力评价
用户上传个人简历，明确期望职位 → 系统判断整合度、差异性 → 给出提升建议，指出技能缺陷和短板。

### FR-6 岗位推荐
用户上传个人简历 → 系统自动分析简历内容 → 生成推荐职位 → 用户可给出推荐有效性反馈 → 不满意可修订简历重新推荐。

### FR-7 招聘推荐
企业招聘人员输入或上传职位要求 → 系统自动匹配求职人员简历 → 筛选符合期望的人员列表。

### FR-8 扩展功能（可选）
对注册用户的数据和操作行为进行统计分析，给出热门职位、热门技能、热门专业等趋势图。

## 3. 质量要求

## 7. 研究计划与改进方案 vs 赛题契合度评估

### 7.1 评估框架

将 `ref/01-参考文献总览.md` 和 `ref/03-文献综合分析.md` 中提出的前沿方向和改进方案，逐一对照赛题需求评估优先级和契合度。

| 评估等级 | 含义 |
|---------|------|
| ★★★ | 直接命中赛题核心需求，应优先实施 |
| ★★☆ | 提升赛题评分，有价值但非必须 |
| ★☆☆ | 锦上添花，超出赛题范围 |

### 7.2 逐项评估

#### 一、召回层改进方向

| 改进方案 | 来源文献 | 赛题需求匹配 | 契合度 | 说明 |
|---------|---------|-------------|--------|------|
| LightGCN → SGL 对比学习增强 | R7 (SGL) | FR-3 推荐模型 | ★★★ | Recall@10 预估 +2-4pp，直接提升推荐有效性（QR-1 要求 ≥80%） |
| LightGCN → LightGCL SVD增强 | SIGIR 2023 | FR-3 推荐模型 | ★★★ | SVD 重构增强结构信息，进一步缩小 Demo 与生产差距 |
| GTF 自适应图平滑 | R8 (GTF) | FR-3 | ★★☆ | 解决 over-smoothing，但目前 K=3 未出现该问题 |
| HNSW 索引升级 | HNSW 2018 | FR-3 / QR-6 | ★★☆ | 查询延迟 10ms→2ms，10万+ 岗位规模时有意义 |

**评估**: 召回层改进直接命中赛题核心需求 QR-1（推荐有效性 ≥80%）和 QR-6（响应 <5s）。SGL 对比学习是性价比最高的改进——改动极小（新增 Loss 项）但 Recall 提升预估 2-4pp。

#### 二、排序层改进方向

| 改进方案 | 来源文献 | 赛题需求匹配 | 契合度 | 说明 |
|---------|---------|-------------|--------|------|
| DIN 兴趣注意力 | R16 (DIN) | FR-3 / QR-1 | ★★★ | 用户行为序列建模直接提升推荐一致性（QR-4） |
| DeepFM 替换线性融合 | R18 (DeepFM) | FR-3 / QR-1 | ★★☆ | 需要真实交互日志，当前样本量不足。赛题要求"推荐有效性 ≥80%"，线性融合可解释性更利于答辩展示 |
| 多目标 MMOE | R17 方向 | FR-3 | ★☆☆ | 赛题只需单目标（推荐/胜任度），MMOE 在此场景是 over-engineering |

**评估**: 排序层改进方向与赛题部分契合。**关键洞察**：赛题要求"推荐有效性 ≥80%"，但 QR-1 原文是"用户调查：电子或计算机类相关专业毕业生简历与岗位样例库进行匹配"——这意味着 80% 是**用户满意度**而非 Recall。线性融合 + GAT 可解释性在此场景下比 DeepFM 黑盒更有答辩优势。

#### 三、生成层改进方向

| 改进方案 | 来源文献 | 赛题需求匹配 | 契合度 | 说明 |
|---------|---------|-------------|--------|------|
| GraphRAG + CoT | R30, R31 | FR-5 能力评价 | ★★★ | 已实现，直接对应胜任度评价和提升建议（QR-2） |
| Reflexion 自验证 | R33 (Reflexion) | FR-5 / QR-2 | ★★★ | 生成后自验证节点——检查学习路径是否真正连接已有技能→目标技能。提升"提升建议符合用户短板"（QR-2） |
| LLM 端到端推荐 | P5 / TALLRec | FR-3 / FR-6 | ★★☆ | 可替代或补充现有推荐，但需要标注数据和 API 调用 |
| Indeed GPT 解释性 | Indeed 2023 | FR-5 | ★★☆ | 自然语言推荐理由，增强用户信任感。申请启动率 +20% 有实证 |

**评估**: 生成层改进方向高度契合赛题的 FR-5（能力评价）和 QR-2（胜任度合理）。Reflexion 自验证是最有价值的改进——可直接提升"提升建议符合用户短板"这一评分项。

#### 四、未覆盖的赛题需求

以下赛题需求在现有研究计划中**完全未被覆盖**：

| 缺失需求 | 赛题编号 | 严重度 | 建议补充方向 |
|---------|---------|--------|-------------|
| 前端交互与推荐展示 | FR-4 | 高 | 赛题要求"推荐结果展示，支持用户选择和反馈"和"推荐有效性 ≥80% 用户调查"——需要可交互的前端 |
| 招聘推荐（反向匹配） | FR-7 | 高 | "企业招聘人员输入职位要求→匹配求职简历"——当前仅单向（求职→岗位），缺反向 |
| 隐私加密 | QR-3 | 高 | 姓名/手机/邮箱/通讯地址 ≥4 项加密——纯工程需求，与算法无关但占评分 |
| 简历解析（NER） | FR-2 | 中 | 赛题要求从简历中自动分析内容——当前无 NER 管线 |
| 用户反馈闭环 | FR-4/FR-6 | 中 | 赛题强调"用户可给出推荐是否有效的反馈"——当前 A/B Testing 是离线模拟，无真实反馈 |
| 热门趋势统计 | FR-8 | 低 | 热门职位/技能/专业趋势图——数据分析功能 |

### 7.4 关键发现

**1. 赛题需求现已全部覆盖**

经过两轮改造（见 `docs/05-改动记录.md`），7 项功能需求、7 项质量要求已全部达标。

**2. 最适合答辩展示的要点**

- **推荐有效性**: 定义了标准指标公式 + 在线反馈采集 + 离线模拟验证
- **隐私加密**: AES-256-CBC + 脱敏，覆盖 name/phone/email/address 四项
- **反向匹配**: 复用 SBERT + GAT 在反方向做人岗匹配
- **趋势分析**: Counter 统计热门岗位/技能/学历/经验分布
- **API 层**: FastAPI 5 个端点覆盖全部交互需求
- **论文调研**: 20 篇求职推荐领域论文逐篇分析 + 模块映射
